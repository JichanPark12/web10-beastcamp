# Ground Rules

## 언어.
- 모든 응답은 한글로 작성합니다.

## 비판적 사고.
- 모든 대답에 좋다고 응답하지 않는다. (아부 금지.)
- 비판적으로 사고하라.

## 점진적 구현.
- 내가 어떤 기능 구현할때 참고한다.
- 한 번에 모든 코드를 작성하지 않고, 단계별로 진행
- 각 단계마다 AI가 만든 코드가 제대로 수행되었는지 사람이 검수하기 위함.
- 절대 모든 과정을 한번에 진행하지 않고 단계별로 끊어서 진행.

## 답변 길이 제한.
- 답변은 한번에 매우 길게 생성하지 마라.
- 장황한 설명보단 핵심적인 요소들로만 간략하게 작성하라.

## 코드 생성은 한개씩.
- 코드 생성은 한번에 여러 파일을 만들지 않는다.
- 만약에 여러 파일을 만들어야한다면, 코드 생성전에 코드 작성 설계 단계를 진행하고 1단계씩만 진행한다.
- 여러 파일들을 동시에 생성하면 안된다.
- 만약 여러 단계를 거쳐야하면 한번씩 끊으며 진행한다.
- 끊는다는 의미는 너가 실행을 종료한다는것이다. (계속 연달아 코드 생성하지 말라는 의미.)

## 변수명은 축약하여.
- 변수명이 매우 장황해지면 안된다.
- 변수명이 매우 길어진다면 축약어를 사용해도 된다.
- 단 변수명에 의도가 확실해야한다.

## 테스트 코드 스타일.
- **프레임워크**: NestJS 기본 테스트 프레임워크와 `supertest`를 사용한다.
- **계층 구조**: `describe`를 활용해 `기능 -> 요청 상황 -> 세부 조건` 순으로 계층화한다.
- **데이터 준비**: DB에 직접 접근하지 않고, `request`를 통해 API로 데이터를 생성(Seed)한다.
- **검증 분리**: `response.status` 검증과 `response.body` 데이터 검증을 별도의 `it` 블록으로 분리한다.
- **한글 명세**: `describe`와 `it`의 설명은 모두 한글로 작성한다. (예: `HTTP 상태 코드 200을 반환해야 한다`)

## Swagger (OpenAPI) 스타일.
- **플러그인 활용**: CLI 플러그인이 설정되어 있으나, 명확한 문서화를 위해 데코레이터를 적극 사용한다.
- **Controller**: `@ApiTags('한글 태그명')`으로 API를 그룹화한다.
- **Method**: `@ApiOperation({ summary: '한글 요약' })`으로 기능을 설명하고, `@ApiResponse`로 응답 타입과 상황을 명시한다.
- **DTO**: `@ApiProperty({ description: '설명', example: '예시값' })`을 사용하여 필드의 용도와 예시 데이터를 구체적으로 제공한다.
- **언어**: 모든 설명과 요약은 한글로 작성한다.

## lint 체크.
- 구현을 했다면 lint를 실행해서 오류가 있는지 확인한다.

## npm 대신 pnpm 사용.
- 프로젝트 루트에 pnpm-lock.yaml과 pnpm-workspace.yaml이 이미 생성되어있다.

## 주석은 최소한.
- 정말 필요한게 아니라면 넣지 않는다.
- 특히 1. 2. 3. 같이 작업 순서를 주석으로 넣지 않는다.
